# Frontend Accessibility

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend accessibility.

## When to use this skill

- When creating new UI components or pages
- When adding interactive elements (buttons, links, forms, modals)
- When implementing keyboard navigation support
- When managing focus states for interactive elements
- When adding ARIA attributes (aria-label, aria-describedby, aria-hidden, etc.)
- When ensuring color contrast meets WCAG AA standards
- When providing alternative text for images, icons, or decorative elements
- When creating form inputs with labels and accessible error messages
- When implementing skip links for keyboard users
- When building modal dialogs with focus management
- When creating accessible dropdown menus or navigation
- When adding screen reader announcements for dynamic content (aria-live)
- When implementing visible focus indicators
- When creating accessible tabs, accordions, or carousels
- When testing components with screen readers (NVDA, VoiceOver)
